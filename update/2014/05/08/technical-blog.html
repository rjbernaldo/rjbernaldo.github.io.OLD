Functional and Object-Oriented Programming

In my opinion, Object-Oriented Programming is more fun to work on than Functional Programming. Take note though that both of these have a purpose and each of them has their own advantages and disadvantages. I believe the best way to show it is through practical examples.

Lets start off with Functional Programming. This is a programming style where you simply code on how you want your script to act on an specific instance.

colors = {:first => "Red", :second => "Blue", :third => "Green"}
p colors
# => {:first => "Red", :second => "Blue", :third => "Green"}
p colors[:first]
# => "Red"

Object Oriented Programming on the other hand considers each part of your code an "Object". This means that each of them will have their own state and methods. For the example below, instead of assigning colors to a value, each color has it's own variable @name (this is called an instance variable) and its own method display (this is called an instance method)

class Color
    def initialize(name)
        @name = name
    end
    def display
        p @name
    end
end

colors = {:first => Color.new("Red"), :second => Color.new("Blue"), :third => Color.new("Green") }

colors[:first].display
# => "Red"

As you can see, each color now has it's own name and display method. Hope I was able to shed some light on the difference of Functional Programming and OOP.
