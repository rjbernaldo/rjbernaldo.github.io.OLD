SQL Issues

Look up issues with SQL (specifically SQL Injection) Enjoy this XKCD comic for an amusing intro. What is it? Why is it a problem? What can newb programmers do to combat this issue?

Basically, SQL injection is a "code injection" technique where malicious SQL statements are inserted into entry fields for execution. According to this code injection cheat sheet, there are various steps to shield your database from the issue mentioned above:

1) Use prepared statements

For SQLite3, it is mentioned that you should use SQL Statement Objects for greater safety. Instead of using dynamic values when asking for queries, each query is prepared before use on the database.

2) Use stored procedures

This is similar to the step above. Basically define the code first, then pass parameters later. The only difference is that the query code itself is stored inside the database to be used later on.

3) Escape all user supplied input

This method forgoes all user supplied input. This method is not recommended. This is mostly used to retrofit legacy code into a newer application.

Compare SQL databases with NoSQL databases. What are the pros and cons of either? Which are you most excited to work with and why?

SQL databases are mostly formatted using a table, and noSQL databases are formatted using the following: document based, key-value pairs, etc. SQL databases have a predefined schema, while noSQL databases do not. SQL databases are vertically scalable, and noSQL databases are horizontally scalable. SQL databases use SQL (Structured Query Language) for defining and manipulating data, noSQL uses an unstructed query to traverse it's collection of data.

I am most excited to work with SQL databases, as I have been been using non-structured databases all my life. I have been accustomed to collecting and manipulating data in my own way, and it has it's big list of faults. I am looking forward to learning more about SQL to eliminate all the problems I have encountered using non-SQL ways of storing data.
